# phase-0005 — User-facing self-service `.ovpn` download

- Status: done
- Depends on: phase-0004
- Goal: Let an assigned user download their own OpenVPN config from their account page, as often as they need, without na-sso ever storing it.
- Done when: a user assigned to an OpenVPN-enabled OPNsense target downloads a `.ovpn` from their account page, a second download returns the same certificate rather than failing or reissuing, every download is audited, and the file is never persisted by na-sso.

## Sub-tasks
1. [done] Add `POST /account/openvpn/{target_id}` returning the config as a file response with `Cache-Control: no-store` and `Pragma: no-cache`, following `na_sso/auth.py:367` — done when: the response headers match and the body is the decoded config.
2. [done] Gate it on the user having an active assignment to that target and on the target's OpenVPN row being enabled and verified — done when: an unassigned user gets 403 and a user on a disabled target sees the control absent.
3. [done] In certificate mode, call `ensure_client_certificate` before export so the cert is minted on first download and reused thereafter — done when: a test shows one certificate created on the first download, none on the second, and the second download returning a config with the same CN. User decision 2026-07-22: re-download is allowed, since OPNsense (not na-sso) holds the key, so blocking it would only generate admin tickets after a device wipe.
4. [done] Surface the control in `account.html` listing each eligible target — done when: the section is absent entirely for a user with no eligible target.
5. [done] Record `openvpn.config_downloaded` audit events including target and mode, and never the config body or key material — done when: a test asserts the audit detail contains no `PRIVATE KEY` substring.
6. [done] Verify end-to-end against the live demo firewall — done when: a real client successfully connects to the demo OpenVPN server using a downloaded config, with the outcome recorded in this Log.
